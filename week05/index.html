Notes

Errors are usually caused by:

  1. System error
  2. Programmer error
  3. User error

Programmer errors are our responsibility, so we must ensure they are minimized as much as possible and fixed promptly

Strict Mode

Not using strict mode is often referred to as 'sloppy mode'. Strict mode encourages a better quality of JavaScript to be written, so its use is recommended.

'use strict';

Error Objects

An "error" object can be created by the host environment when an exception occurs, or it can be created in the code using a constructor function.

const error = new Error('Oops, something went wrong');

How many error objects are there?

There are seven more error objects used for specific errors:

  • EvalError, not used in the current ECMAScript.
  • RangeError, thrown when a number is outside an allowable range of values.
  • ReferenceError, thrown when a reference is made to an item that doesn’t exist.
  • SyntaxError, thrown when there’s an error in the code’s syntax.
  • TypeError, thrown when there’s an error in the type of value used.
  • URIError, thrown when there’s a problem encoding or decoding the URI
  • InternalError, a non-standard error that is thrown when an error occurs in the JavaScript engine.
