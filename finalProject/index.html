Description:

My application is basically a movie search engine. Through my application you can quickly search for a desired movie, using the full or partial title. Once the films have been listed according to what was searched, more information can be obtained by clicking on their posters. It also has a function to save your favorite movies.

API:

The API I am using is Movie Database (IMDB Alternative). Endpoints return JSON with a lot of useful information.

JavaScript:

Something that I am using that has helped me keep the logic tidy is the MVC design pattern. From the model I return the information in JSON to the controller, and then render the information using the view. The main.js file is the entry point to my project, where I have only created an instance of the controller that starts when the window is loaded

LocalStorage:

It is used to save the movies

Design Process:

I had previously done a similar project using the PokeAPI, so I tried to use the same layout to display the information. In the process I had to make several changes to make the experience more fluid, that is why I used a modal to show the detailed information of the movie.

Problems Solved:

  • I had some trouble returning the resolved data from the fetch function. I was able to fix it by retorting the fetch function, and using the promises.
